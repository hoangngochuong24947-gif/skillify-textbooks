# 05-主技能规范：《示例书名》

## 元数据

```yaml
name: example-book-thinking
description: Use when applying [示例书名] thinking to user problems. Triggers on questions about [domain keywords].
source_book:
  title: 示例书名
  author: 示例作者
  domain: 决策心理学
version: "1.0.0"
platform_compatibility:
  - Claude Code
  - OpenClaw
```

## 顶层调用目标

当用户提出问题时，本 skill 帮助判断：
1. 是否应该使用《示例书名》的思维方式？
2. 哪些方法论模块最相关？
3. 应该调用哪些 subagents？
4. 将产出哪些输出文档？

## 进入条件

### 自动触发

当用户输入包含以下关键词或模式时自动触发：
- "用《示例书名》的思维..."
- "根据示例作者的方法..."
- "如何像书中说的那样..."
- 问题涉及：[列出关键词领域]

### 手动调用

用户明确请求：
- "帮我用这本书分析..."
- "从这本书的角度看..."

## 核心能力

### 能力一：问题适配性判断

**功能：** 判断用户问题是否适合用本书思维分析

**输入：** 用户问题文本
**输出：** `RoutingDecision` 或 `null`（不适合）

**判断逻辑：**
```
IF 问题类型 IN [适配类型列表] THEN
    返回匹配的方法和 agents
ELSE IF 问题类型 IN [模糊类型列表] THEN
    返回 low confidence + 澄清问题
ELSE
    返回 null + 推荐替代资源
```

### 能力二：方法筛选

**功能：** 从 MethodCards 中选择最匹配的方法

**筛选规则：**
1. 匹配 triggers 字段
2. 检查 limits 字段，排除不适用的
3. 按证据强度排序
4. 返回 top-3 方法

### 能力三：Subagent 委派

**功能：** 根据问题类型决定调用哪些 subagents

**委派规则：**

| 问题特征 | 委派 Subagents | 输出文档 |
|----------|----------------|----------|
| 需要理解全书框架 | Book Profiler | 00-overview.md |
| 需要作者视角 | Author Thinking Analyst | 01-author-thinking.md |
| 需要具体方法 | Methodology Miner | 02-method-catalog.md |
| 需要场景匹配 | Scenario Router | 03-scenario-router.md |
| 完整 skill 包 | 全部 | 全部文档 |

## 执行流程

### 标准流程

```
1. 接收用户问题
   ↓
2. 判断适配性
   ├─ 不适合 → 礼貌拒绝，推荐替代
   └─ 适合 → 继续
   ↓
3. 筛选方法
   ↓
4. 委派 Subagents
   ↓
5. 并行/串行执行
   ↓
6. 整合输出
   ↓
7. 返回结果 + 引用证据
```

### 快速流程（适用于简单问题）

```
1. 接收用户问题
   ↓
2. 直接匹配 RoutingDecision 缓存
   ↓
3. 返回预设答案 + 证据引用
```

## 证据与降级规则

### 证据强度分级

| 级别 | 标准 | 处理方式 |
|------|------|----------|
| Strong | 直接引用 + 章节位置 | 正常输出 |
| Medium | 间接推断 + 多处支持 | 正常输出，标记推断 |
| Weak | 单次提及或隐含 | 降级为 pending |
| None | 无证据 | 拒绝输出，记录缺口 |

### 降级规则

- **Strong → Medium：** 保留，添加推断标记
- **Medium → Weak：** 移至待确认清单
- **Weak → None：** 不纳入 skill，记录缺口

### 回补提问触发

当以下情况发生时，触发回补提问：
- MethodCard 证据字段为空
- 用户问题需要未覆盖的方法
- 验证发现理解偏差

### 适用判断
[是否适用及理由]

### 推荐方法
1. [方法名称]
   - 核心论点：...
   - 适用步骤：...
   - 证据来源：Chapter X

### 执行建议
[具体操作建议]

### 边界提醒
[不适用的情况提醒]

### 引用格式

所有结论必须附带证据：
- 直接引用：`书中指出："引用内容"（Chapter X, Section Y）`
- 推断标注：`基于 Chapter X 推断：[结论]`

## 常见问答

### Q1: 这本书适合解决我的问题吗？

**判断标准：**
- 如果你的问题是 [类型A]，那么适合
- 如果你的问题是 [类型B]，部分适合
- 如果你的问题是 [类型C]，不适合，建议 [替代方案]

### Q2: 为什么推荐使用这个方法而不是那个？

**解释模板：**
- 根据 Chapter X，方法A适用于 [条件]
- 你的问题符合这些条件
- 方法B的 triggers 要求 [条件]，你的问题不满足

### Q3: 证据不足时怎么办？

**处理流程：**
1. 明确告知证据不足
2. 说明当前的推断依据
3. 建议进行回补提问
4. 或推荐其他更可靠的资源
